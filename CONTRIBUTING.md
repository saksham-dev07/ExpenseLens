# Contributing to Advanced OCR Receipt Scanner

First off, thank you for considering contributing to the Advanced OCR Receipt Scanner! It's people like you that make open-source software such a great community.

## Code of Conduct
By participating in this project, you are expected to uphold a welcoming and respectful environment for everyone.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.
- **Use the Bug Report Template:** Check the `.github/ISSUE_TEMPLATE/bug_report.md` file when creating a new issue.
- **Provide clear reproduction steps:** Explain exactly what you did that caused the bug.
- **Include environment details:** OS, browser, Python version, Node version, etc.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.
- **Use the Feature Request Template:** Check the `.github/ISSUE_TEMPLATE/feature_request.md` file.
- **Provide a compelling reason:** Explain why this enhancement would be useful to most users.

### Pull Requests
The process described here has several goals:
- Maintain project quality.
- Fix problems that are important to users.
- Engage the community in working toward the best possible application.

Please follow these steps to have your contribution considered by the maintainers:
1. **Fork the repo** and create your branch from `main`.
2. If you've added code that should be tested, **add tests**.
3. Ensure the test suite passes.
4. Make sure your code lints.
5. **Issue that PR!**

## Styleguides

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally after the first line.

### Python Styleguide
- All Python must adhere to [PEP 8](https://peps.python.org/pep-0008/).
- Use descriptive variable names.
- Provide docstrings for all functions and classes.

### React/TypeScript Styleguide
- Use functional components and React Hooks.
- Prefer Tailwind CSS for styling.
- Keep components small and focused.
